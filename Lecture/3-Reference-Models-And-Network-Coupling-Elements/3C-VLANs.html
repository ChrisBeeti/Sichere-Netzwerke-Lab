VLANs — Virtual Local Area Networks
Virtuelle Unterteilung eines physischen Netzwerks
Broadcast-Domänen — Vergleich
Klick auf ▶ zum Starten
⚙️ Funktion
Geräte werden auf Layer 2 in separate, voneinander isolierte Broadcast-Domänen aufgeteilt — unabhängig von ihrer physischen Verkabelung.
🎯 Zweck
Logische Gruppierung von Geräten nach Abteilung, Funktion oder Sicherheitsanforderung — unabhängig vom physischen Standort im Gebäude.
Technisch
Geräte im gleichen VLAN kommunizieren direkt, als wären sie im gleichen physischen LAN. Kommunikation zwischen VLANs erfordert einen Layer-3-Router oder einen Layer-3-Switch.
Vorteile
✓ Reduktion von Broadcast-Traffic
✓ Sicherheitsisolierung
✓ Flexibles Netzwerkdesign
✓ Bessere Skalierbarkeit
Typische Anwendung
🏢 Unternehmen: VLAN je Abteilung
🔒 DMZ: Server isoliert von Clients
📶 WLAN: Gäste-VLAN getrennt
🏭 Industrie: OT-Netz isoliert

IEEE 802.1Q — VLAN-Tagging
Wie VLANs über Switches und Trunk-Leitungen realisiert werden
VLAN wählen → Paket senden
Access Port
Gehört zu einem VLAN. Verbindet Endgerät mit Switch. Frame hat kein Tag (untagged) — das VLAN wird am Port konfiguriert.
Trunk Port
Überträgt mehrere VLANs gleichzeitig. Frames erhalten ein 802.1Q-Tag (4 Byte) mit VLAN-ID — für Switch-zu-Switch-Verbindungen.
